# Navigator: ручной ввод «моей позиции» (адрес + булавка на карте)

**Дата:** 2026-05-23
**Раздел:** Crypto Navigator (`/{locale}/coin/{slug}#navigator`)

## Задача

Сейчас позицию пользователя (origin маршрута) можно задать двумя способами:
📍 авто-геолокацией и полем адреса (Nominatim-геокодинг — уже работает, но не
подписано явно как «моя позиция»). Не хватает третьего, наглядного способа —
**поставить булавку прямо на карте**. Клик по пустому месту карты сейчас
ничего не делает.

## Подход

Свести все три способа в один блок «Моя позиция» и добавить булавку кликом:

- **Блок «Моя позиция»** — заголовок + строка: `[📍 Авто]` + поле адреса
  (то же, что и сейчас) + `[Найти]`; ниже подсказка «или кликните по карте»;
  при заданной позиции — чип текущей позиции (адрес / «Точка на карте» / GPS)
  с крестиком сброса.
- **Булавка кликом** — клик по пустому месту карты ставит оранжевую
  **перетаскиваемую** булавку = origin. Перетащил → позиция обновилась,
  маршрут (если выбран пункт назначения) перестраивается. Клик по POI
  по-прежнему открывает карточку: в общем обработчике клика отсекаем точки
  через `map.queryRenderedFeatures(point, { layers: poi-* })`.
- **Обратный геокодинг (best-effort)** — после установки/перетаскивания
  булавки запрашиваем Nominatim `reverse`, чтобы показать адрес в чипе. Нет
  ответа — остаётся «Точка на карте».

Всё key-less, в духе проекта (тот же хост Nominatim, что и для прямого поиска).

## Изменения в коде

### `src/lib/crypto-map.ts`
- `NOMINATIM_REVERSE_URL = "https://nominatim.openstreetmap.org/reverse"`.
- `parseReverseGeocode(raw): GeoResult | null` — чистая: reverse отдаёт один
  объект `{ display_name, lat, lon }` (а не массив, как `search`); при
  `{ error }` или нечисловых координатах → `null`. **Юнит-тест.**
- `fetchReverseGeocode(lon, lat, timeoutMs = 8000): Promise<GeoResult | null>`.

### `src/app/api/crypto-map/geocode/route.ts`
- Если есть `&lat=&lon=` → reverse-ветка: валидация диапазонов, кэш
  `cmap:rgeo:{lat.4},{lon.4}` (TTL 1d), ответ `{ result: GeoResult | null }`.
  Прямой поиск (`?q=`) не меняется (`{ results: [...] }`).

### `src/components/coin-detail/crypto-navigator.tsx`
- Состояние `originLabel: string | null`.
- Хелпер `setOrigin(lonlat, label, reverse)` — единая точка установки origin:
  ставит/двигает перетаскиваемую оранжевую булавку, на `dragend` обновляет
  origin + reverse-лейбл + перестраивает маршрут; затем `buildRoute()` если
  есть пункт назначения. Используется из поля адреса, клика по карте и GPS.
- `reverseLabel(lonlat)` — ставит «Точка на карте», затем подменяет адресом из
  reverse-эндпоинта.
- Общий `map.on("click")`: пропускает клики по POI-кружкам, иначе
  `setOrigin(lngLat, null, reverse=true)`.
- GPS-обработчик: убирает оранжевую булавку (у контрола свой синий маркер),
  ставит лейбл GPS.
- `submitSearch` переписан через `setOrigin(..., reverse=false)`.
- `clearOrigin()` — снимает булавку, обнуляет origin/лейбл, очищает линию
  маршрута.
- Блок «Моя позиция» в разметке (заменяет отдельную форму адреса); 📍-кнопка
  переезжает из верхнего ряда контролов в этот блок.

### `messages/*.json` (10 локалей)
Новые ключи в `cryptoMap`: `myPosition`, `posAuto`, `clickToDrop`, `posPin`,
`posGps`, `clearOrigin`.

## Краёвые случаи / деградация
- Reverse недоступен → чип показывает «Точка на карте» (без падения).
- Клик по POI не двигает origin (фильтр по `queryRenderedFeatures`).
- Сброс origin при выбранном пункте назначения → линия маршрута убирается.
- Обратная совместимость API: вызов `?q=` ведёт себя как раньше.

## Тестирование
- Юнит: `parseReverseGeocode` (валидный объект → GeoResult; `{error}` → null;
  нечисловые/пустые → null). Остальные парсеры без изменений.
- Сборка + typecheck. Ручная проверка: клик по карте ставит булавку,
  перетаскивание двигает её и перестраивает маршрут, чип показывает адрес.
